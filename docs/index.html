Why do I keep getting "a network error occurred"?

Janitor AI shows the same generic message for almost every proxy failure. Underneath, the cause is nearly always one of these:

  • Mismatched pieces: an API key from one provider with a URL from another (the #1 cause). A DeepSeek key cannot talk to openrouter.ai and vice versa.
  • Wrong URL shape: the field needs the full endpoint ending in /chat/completions, not the base address, and it must be https.
  • Stale model names: tutorials age fast. DeepSeek's "deepseek-chat" and "deepseek-reasoner" stop working on July 24, 2026; OpenRouter needs vendor/model format.
  • Money, not settings: DeepSeek has no free tier (402 Insufficient Balance on fresh accounts), OpenRouter caps free models at 50 requests/day, Gemini's free quota resets daily.
  • Hidden characters: mobile copy-paste loves to add spaces and invisible characters that make an otherwise perfect key fail.

Is it safe to paste my API key here?

Yes. ProxyMedic is a static page with no server and no analytics. The static checks run entirely on your device. If you choose to run the live test, your browser talks directly to your own provider (the same thing Janitor does when you chat), and the key is never stored, logged, or sent anywhere else. The page is open source, so you can verify that.

Which providers does it know?

OpenRouter, DeepSeek, Google Gemini, Chutes, Groq, Mistral, OpenAI and Anthropic are recognised with full checks (key format, exact endpoint, live model list). Community reverse proxies are supported through the live test.
